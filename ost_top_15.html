1. Compare the advantages and disadvantages of open-source and closed-source software

  • Open‑Source Software

    1. Cost: Generally free to use and distribute; eliminates licensing fees.

    2. Flexibility: Source code access lets you tailor features to your needs.

    3. Community Support: Large user/developer communities share fixes, plugins, documentation.

    4. Security: “Many eyes” on the code often lead to rapid discovery and patching of vulnerabilities.

    5. Freedom: Unrestricted redistribution and modification under open licenses.

  • Drawbacks of Open‑Source

    1. Support Variability: No guaranteed commercial support; you may rely on forums or pay third parties.

    2. Usability: Interfaces and installers can be less polished than commercial equivalents.

    3. Compatibility: May not integrate smoothly with proprietary systems or file formats.

    4. Feature Gaps: Niche or highly specialized features may be missing.

    5. Fragmentation: Multiple forks/distributions can confuse users choosing the “right” version.

  • Closed‑Source Software

    1. Vendor Support: Professional helpdesks, SLAs, training, and detailed documentation.

    2. Polish & Stability: Rigorous testing and unified release cycles often yield user‑friendly UIs.

    3. Feature Completeness: Developers prioritize market‑driven features and long‑term roadmaps.

  • Drawbacks of Closed‑Source

    1. Costly Licensing: Upfront fees, subscriptions, and upgrade costs.

    2. Lack of Transparency: Cannot audit or modify code; security relies entirely on vendor.

    3. Vendor Lock‑In: Customization limited; migrating away may be difficult or expensive.

2. Explain how PHP code can be embedded within an HTML file with an example

  • Mechanism: Save your file with a .php extension. Surround PHP with <?php … ?> tags anywhere in the HTML.

  • Parser Behavior: The server executes PHP blocks and replaces them with their output before sending HTML to the browser.

  • Example:

      <!DOCTYPE html>
  <html>
    <head><title>My PHP Page</title></head>
    <body>
      <h1><?php echo "Welcome to my site!"; ?></h1>
      <p>Today is <?php echo date("l, F j, Y"); ?>.</p>
    </body>
  </html>

  • Scope Sharing: Variables declared in one PHP block are available in subsequent blocks in the same file.

  • Best Practice: Minimize logic in views; keep complex code in separate scripts or functions.

3. Write a PHP script to collect form data using POST and display it on another page

  1. HTML Form (form.html):

     <form action="result.php" method="POST">
   <label>Name: <input type="text" name="username"></label><br>
   <label>Email: <input type="email" name="email"></label><br>
   <button type="submit">Submit</button>
 </form>

  2. Processing Script (result.php):

     <?php
   // Always validate/sanitize in production!
   $name  = htmlspecialchars($_POST['username']);
   $email = htmlspecialchars($_POST['email']);
   echo "<h2>Form Submission</h2>";
   echo "<strong>Name:</strong> $name<br>";
   echo "<strong>Email:</strong> $email";
 ?>

  3. How It Works:

    • The browser sends form fields in the HTTP request body.

    • PHP’s $_POST superglobal holds the data.

    • Use htmlspecialchars() to prevent XSS when echoing user input.

  4. Security Tip: Validate server‑side (e.g., check email format with filter_var).

5. Explain the role of prepared statements in preventing SQL injection

  1. Separation of Code and Data: Prepared statements send SQL with placeholders first; data is bound later.

  2. Parameter Binding: User input is never concatenated into SQL strings, so it cannot break out of intended context.

  3. Automatic Escaping: The database driver treats bound values purely as data, escaping special characters.

  4. Example (PDO):

     $sql  = "SELECT * FROM users WHERE email = :email";
 $stmt = $pdo->prepare($sql);
 $stmt->execute(['email' => $_GET['email']]);
 $user = $stmt->fetch();

  5. Benefit: Even if $_GET['email'] contains malicious payload, it cannot alter the query structure.

6. List and explain five important array functions in PHP

  1. array_merge()

    • What: Joins multiple arrays end‑to‑end.

    • Use: Combine data from different sources.

    • Example: array_merge([1,2], [3,4]) → [1,2,3,4].

  2. array_push() / array_pop()

    • Push: Appends elements to end.

    • Pop: Removes and returns the last element.

    • Example:

        $stack = ["a","b"];
  array_push($stack, "c"); // ["a","b","c"]
  $last = array_pop($stack); // "c"

  3. array_shift() / array_unshift()

    • Shift: Removes the first element.

    • Unshift: Prepends one or more elements.

    • Example:

        $q = ["x","y"];
  array_unshift($q, "z"); // ["z","x","y"]
  $first = array_shift($q); // "z"

  4. in_array()

    • What: Checks if a value exists in an array. Returns boolean.

    • Use: Validate user input against allowed values.

    • Example: in_array("apple", ["apple","banana"]) → true.

  5. array_key_exists()

    • What: Determines if a given key exists in an array (especially for associative arrays).

    • Example:

        $user = ["name"=>"Bob"];
  array_key_exists("name", $user); // true

7. Compare static and dynamic websites with real-life examples

  • Static Websites

    • Definition: Each page is a fixed HTML file served directly.

    • Example: A simple portfolio site with “About,” “Projects,” and “Contact” pages.

    • Pros: Fast load times, low server load, easy to host.

    • Cons: Content updates require manual editing and redeployment.

  • Dynamic Websites

    • Definition: Pages generated on the server (e.g., via PHP), often querying a database.

    • Example: An online store where product listings, prices, and user carts update in real time.

    • Pros: Personalized content, easy content management via CMS.

    • Cons: More complex hosting, potential performance overhead under high traffic.

Feature Static Dynamic
Updates Manual file changes Through admin panels or scripts
Interactivity Limited (client‑side JS only) High (user sessions, forms, search)
Scalability Scale via CDNs easily Requires server scaling & caching

8. Describe the steps to handle file uploads securely in PHP

  1. HTML Form:

     <form action="upload.php" method="POST" enctype="multipart/form-data">
   <input type="file" name="fileupload">
   <button type="submit">Upload</button>
 </form>

  2. Check Upload Success:

     if (!isset($_FILES['fileupload']) || $_FILES['fileupload']['error'] !== UPLOAD_ERR_OK) {
   exit("Upload failed.");
 }

  3. Validate File Type:

     $ext       = strtolower(pathinfo($_FILES['fileupload']['name'], PATHINFO_EXTENSION));
 $allowed   = ['jpg','png','gif'];
 if (!in_array($ext, $allowed)) exit("Invalid file type.");

  4. Limit File Size:

     if ($_FILES['fileupload']['size'] > 2_000_000) exit("Max 2 MB allowed.");

  5. Generate Safe Filename:

     $safeName  = uniqid() . '.' . $ext;
 $targetDir = __DIR__ . '/uploads/';

  6. Move Uploaded File:

     if (!move_uploaded_file($_FILES['fileupload']['tmp_name'], $targetDir . $safeName)) {
   exit("Error saving file.");
 }
 echo "Upload successful!";

  7. Store Metadata: Optionally record $safeName and upload path in a database for tracking.

9. Explain the difference between include and require in PHP with examples

  • include 'file.php';

    • Executes the specified file.

    • Emits a warning on failure but continues script execution.

    • Use: Non‑critical components (e.g., optional templates).

  • require 'file.php';

    • Executes the specified file.

    • Emits a fatal error on failure and halts execution.

    • Use: Essential dependencies (e.g., database configuration).

  • Example:

      include 'sidebar.php';  // If missing, page still loads without sidebar
  require 'config.php';   // Must exist, or the script stops immediately

10. Write a PHP script to demonstrate CRUD operations using MySQL

<?php
// 1. Connect using PDO
$pdo = new PDO("mysql:host=localhost;dbname=testdb;charset=utf8mb4", "root", "");

// 2. CREATE
$insert = $pdo->prepare("INSERT INTO contacts (name,email) VALUES (:n,:e)");
$insert->execute(['n'=>'Alice','e'=>'alice@example.com']);

// 3. READ
$stmt = $pdo->query("SELECT id,name,email FROM contacts");
$all   = $stmt->fetchAll();
foreach ($all as $row) {
  echo "{$row['id']}: {$row['name']} ({$row['email']})<br>";
}

// 4. UPDATE
$update = $pdo->prepare("UPDATE contacts SET email = :e WHERE id = :i");
$update->execute(['e'=>'newalice@example.com','i'=>1]);

// 5. DELETE
$delete = $pdo->prepare("DELETE FROM contacts WHERE id = ?");
$delete->execute([1]);
?>
  • Use try-catch and set PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION for robust error handling.

11. Discuss the importance of sessions in PHP and demonstrate the use of $_SESSION

  • Why Sessions? HTTP is stateless; sessions let you persist data (e.g., login status) across multiple requests.

  • Starting a Session: session_start(); must be called before any output.

  • Storing Values:

      $_SESSION['user_id']   = $userId;
  $_SESSION['last_visit']= time();

  • Retrieving Values:

      echo "Welcome back, user #" . $_SESSION['user_id'];

  • Ending a Session:

      session_unset();    // Clear all session vars
  session_destroy();  // End session and delete cookie

  • Use Cases: Shopping carts, user authentication, multi‑step forms.

12. Explain how conditional structures (if-else, switch) are used in PHP with real-world applications

  • if-else: Execute code based on boolean conditions.

      if ($age >= 18) {
    echo "Access granted";
  } else {
    echo "Must be 18+";
  }

  • elseif: Chain multiple conditions.

  • Real‑World: Form validation (required fields, value ranges).

  • switch: Simplify multiple comparisons on the same variable.

      switch ($role) {
    case 'admin':   $layout = 'admin.tpl'; break;
    case 'editor':  $layout = 'editor.tpl'; break;
    default:        $layout = 'guest.tpl';
  }

  • Real‑World: Routing requests based on query parameters or command tokens.

13. Compare associative arrays and indexed arrays in PHP with examples

  • Indexed Arrays

    • Keys are numeric, starting at 0 by default.

    • Example:

        $colors = ["Red","Green","Blue"];
  echo $colors[2]; // Blue

  • Associative Arrays

    • Keys are user‑defined strings.

    • Example:

        $user = ["id"=>42,"name"=>"Alice"];
  echo $user['name']; // Alice

  • Use Cases:

    • Indexed for ordered lists.

    • Associative for mapping meaningful labels to values.

14. What are cookies? Write a PHP script to set, retrieve, and delete a cookie

  • Definition: Small files stored on the client by the browser; hold stateful data across sessions.

  • Set a Cookie:

      setcookie("theme","dark", time()+86400*7, "/"); // Expires in 7 days

  • Retrieve a Cookie:

      if (isset($_COOKIE['theme'])) {
    echo "Current theme: " . $_COOKIE['theme'];
  }

  • Delete a Cookie:

      setcookie("theme","", time()-3600, "/"); // Expire in past

  • Notes: Must call setcookie() before any HTML output.

15. Explain the role of the header() function in PHP and demonstrate its use for redirection

  • Purpose: Sends raw HTTP headers to the client before any body output.

  • Redirection Example:

      header("Location: /dashboard.php");
  exit;  // Prevent further script execution

  • Other Uses:

    • Set content type: header("Content-Type: application/json");

    • Control caching: header("Cache-Control: no-cache");

    • File downloads:

        header("Content-Disposition: attachment; filename=\"report.pdf\"");
  readfile("report.pdf");
  exit;

  • Important: Always call before any echo or HTML.
