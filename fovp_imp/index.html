Unit 1: Introduction to .NET Framework

1. Define the Common Language Runtime (CLR) in the .NET Framework. Explain its role in the execution of .NET applications.

The Common Language Runtime (CLR) is the "heart" or "core component" of the .NET Framework. It is the execution engine that manages running applications.

Key Roles:

  • Code Execution: It loads and executes .NET applications.
  • JIT Compilation: It converts the intermediate code (MSIL) into native machine code (code the CPU understands) using a Just-In-Time (JIT) compiler.
  • Memory Management: It automatically manages memory using the Garbage Collector (GC), which frees memory from objects that are no longer needed.
  • Security: Enforces code access security (CAS) to prevent unauthorized operations.
  • Exception Handling: Provides a standard way to handle errors (exceptions) across different languages.
  • Type Safety: Enforces type rules via the Common Type System (CTS).

2. What is the Common Type System (CTS) in .NET? How does it contribute to the language interoperability in .NET?

The Common Type System (CTS) describes the set of all data types that can be used in different .NET languages.

Contribution to Interoperability:

The CTS ensures that data types defined in one language can be understood by another. It provides a common standard.

  • Example: A int in C# and an Integer in VB.NET both map to the same CTS type: System.Int32.
  • Because both languages use this common type, they can easily interact, pass data to each other, and even inherit classes from one another.

3. Explain the significance of Garbage Collection in .NET. How does it improve memory management?

Garbage Collection (GC) is the .NET Framework's automatic memory manager. Its significance is that it frees programmers from having to manually allocate and de-allocate memory.

How it improves memory management:

  • Prevents Memory Leaks: The GC periodically runs and checks for "unreferenced objects" (objects in memory that the application is no longer using).
  • Reclaims Memory: It automatically reclaims the memory used by these objects, making it available for new objects.
  • Increases Stability: By handling memory automatically, it reduces common programming errors (like "dangling pointers" or forgetting to free memory) that lead to application crashes.

4. Explain the differences between managed code and unmanaged code.

  • Managed Code:
    • This is code that runs *inside* the .NET Common Language Runtime (CLR).
    • Examples: C#, VB.NET.
    • Benefits: It gets all the services from the CLR, such as automatic garbage collection, type safety, and security.
    • It is first compiled to MSIL (Microsoft Intermediate Language).
  • Unmanaged Code:
    • This is code that runs *outside* the CLR, directly on the operating system (CPU).
    • Examples: C, C++.
    • Responsibilities: The programmer is fully responsible for all memory management (allocating and freeing memory), type safety, and security.

5. Evaluate the role of the CLR's Just-In-Time (JIT) compilation approach.

The JIT (Just-In-Time) compiler is a key part of the CLR's execution model. It balances portability with performance.

Execution Process:

  1. Compile-Time: Your C# code is compiled into MSIL (Microsoft Intermediate Language). This MSIL is platform-independent.
  2. Run-Time: When you run the application, the CLR's JIT compiler loads the MSIL.
  3. JIT Compilation: Right before a method is executed *for the first time*, the JIT compiler converts that method's MSIL into native machine code (code the CPU can execute directly).
  4. Caching: This native code is then cached in memory. The *next* time that method is called, the CLR executes the cached native code directly, skipping compilation.

Advantages:

  • Portability: The same MSIL file can run on any device (Windows, Linux, Mac) that has a compatible .NET CLR.
  • Performance: It's faster than pure interpretation because the code is compiled to native. It can also optimize code at runtime based on the specific machine.

Unit 2: Basics of Object-Oriented programming using C#

1. List and describe the four main principles of Object-Oriented Programming.

  1. Encapsulation: The practice of binding data (variables/fields) and the methods (functions) that operate on that data into a single unit, called a class. It hides the internal complexity from the outside world (e.g., using private fields) and exposes only necessary functionality (e.g., using public methods).
  2. Inheritance: The mechanism that allows a new class (child class or derived class) to "acquire" or "inherit" the properties and methods of an existing class (parent class or base class). This promotes code reusability.
  3. Polymorphism: (Meaning "many forms"). The ability for methods or objects to take on different forms.
    • Method Overloading (Compile-Time): Having multiple methods with the *same name* but *different parameters* (different type or number of arguments).
    • Method Overriding (Run-Time): A child class provides its *own specific implementation* of a method that is already defined in its parent class.
  4. Abstraction: Hiding the complex implementation details and showing only the essential features of an object. It focuses on *what* an object does, not *how* it does it. (e.g., you press the 'on' button on a TV, you don't need to know how the circuits work).

2. What are Boxing and Unboxing in C#? Provide examples.

Boxing and Unboxing are processes that allow Value Types and Reference Types to work together.

  • Boxing: The process of converting a Value Type (like int, double, struct) into a Reference Type (like object). This is an implicit (automatic) conversion. The value is "boxed" and stored on the heap.
  • Unboxing: The process of converting the Reference Type (object) back into a Value Type (int). This is an explicit (manual) conversion and requires a cast.
// 1. A Value Type
int myValue = 123;

// 2. Boxing: myValue (on the stack) is "boxed" 
// and copied to myObject (on the heap). This is implicit.
object myObject = myValue;

// 3. Unboxing: myObject is "unboxed" back to
// an int. This is explicit and requires a cast.
int myUnboxedValue = (int)myObject;

3. Explain the concept of Checked and Unchecked Blocks in C#. Why are these important for handling overflow?

These blocks control how C# handles arithmetic overflow for integer types (e.g., an int value going above 2,147,483,647).

  • checked block:
    • This block *enforces* overflow checking at runtime.
    • If an operation inside this block overflows, it will throw an OverflowException.
    • Use this when you need to ensure your calculations are arithmetically correct and stop the program if they are not.
  • unchecked block:
    • This block *disables* overflow checking.
    • If an operation overflows, the value is simply truncated (it "wraps around," usually to a large negative number) and no exception is thrown.
    • This is the default behavior in C# for most operations.
int maxInt = int.MaxValue;

try
{
    // This will THROW an exception
    checked
    {
        int x = maxInt + 1; 
    }
}
catch (OverflowException ex)
{
    Console.WriteLine("Checked block overflowed!");
}

// This will NOT throw. 'y' will be -2147483648
unchecked
{
    int y = maxInt + 1;
}

4. How does C# handle type casting? Differentiate between implicit and explicit casting with examples.

Type casting is converting a variable from one data type to another.

  • Implicit Casting (Automatic):
    • Done automatically by the C# compiler.
    • This is only allowed when the conversion is safe and there is no risk of data loss.
    • This typically involves converting from a "smaller" type to a "larger" type (e.g., int to double).
    • Example: int myInt = 100; double myDouble = myInt; // Automatic
  • Explicit Casting (Manual):
    • Must be done manually by the programmer using the cast operator (type).
    • This is required when there is a risk of data loss (e.g., loss of precision or magnitude).
    • This typically involves converting from a "larger" type to a "smaller" type (e.g., double to int).
    • Example: double myDouble = 9.78; int myInt = (int)myDouble; // Manual cast. myInt is now 9

5. Explain how a multi-dimensional array is declared and used in C#. Also, explain the concept of jagged arrays.

  • Multi-dimensional Array (Rectangular):
    • An array with more than one dimension, fixed in a grid-like (table) structure where each row has the same number of columns.
    • Declaration: datatype[,] arrayName = new datatype[rows, columns];
    • Example: int[,] matrix = new int[3, 3];
    • Usage: matrix[0, 1] = 5; // Access row 0, column 1
  • Jagged Array (Array of Arrays):
    • An array whose elements are *other arrays*.
    • The key difference is that each inner array can have a different length.
    • Declaration: datatype[][] arrayName = new datatype[rows][];
    • Example: 
      int[][] jagged = new int[3][]; // An array of 3 arrays
jagged[0] = new int[2]; // Row 0 has 2 elements
jagged[1] = new int[5]; // Row 1 has 5 elements
jagged[2] = new int[3]; // Row 2 has 3 elements
jagged[1][3] = 10; // Access row 1, element 3

Unit 3: Building desktop application (Windows Forms)

1. What is the role of the MessageBox class in Windows Forms?

The MessageBox class is used to display a pre-built dialog box (a popup window) to the user. It is a modal window, which means the user must interact with it (click a button) before they can return to the main application.

Its roles are:

  • To Show Information: Display a simple message to the user (e.g., "Operation Complete").
  • To Ask Questions: Get a simple response from the user (e.g., "Are you sure you want to delete?") by showing buttons like YesNo, YesNoCancel, or OKCancel.
  • To Show Errors/Warnings: Display a message with a specific icon (like Error, Warning, Information) to alert the user.

You don't create an instance of it. You use the static Show() method:
MessageBox.Show("Hello World", "My Caption", MessageBoxButtons.OK, MessageBoxIcon.Information);

3. Explain the role of the ErrorProvider control in Windows Forms. How does it help in form validation?

The ErrorProvider is a non-visual component that provides a user-friendly way to show validation errors.

Role & How it Helps:

  1. You drag an ErrorProvider from the Toolbox onto your form (it sits in the component tray).
  2. When you validate user input (e.g., in a button click or a control's Validating event), you check if the data is bad.
  3. If input is invalid: You call the SetError() method.
    errorProvider1.SetError(textBox1, "Name cannot be empty!");
  4. This makes a small, blinking error icon (e.g., !) appear next to textBox1. When the user hovers over the icon, they see the error message ("Name cannot be empty!") as a tooltip.
  5. If input is valid: You clear the error by passing an empty string.
    errorProvider1.SetError(textBox1, "");

It's better than a MessageBox because it doesn't interrupt the user's workflow and can show multiple errors on the form at once.

4. Evaluate the use of container controls (like Panel and GroupBox) in organizing controls in a Windows Form.

Container controls are used to group other controls together. This is essential for layout management and a clean UI.

  • GroupBox:
    • Purpose: Visually groups related controls.
    • Features: It has a Text property that creates a caption and draws a border around the controls.
    • Main Use: Its most important functional use is to group RadioButtons. Only one RadioButton inside a GroupBox can be selected at a time.
  • Panel:
    • Purpose: A more general-purpose container for grouping controls.
    • Features: It's invisible by default (no border). Its main power comes from:
      • Movement: Moving the Panel moves all controls inside it.
      • Visibility: Setting panel1.Visible = false hides the panel and *all* its child controls.
      • Scrolling: Can provide scrollbars (AutoScroll = true) if the content inside is too large.

5. Understanding the Life-cycle of a Form.

The form life-cycle is the sequence of events that are raised from the moment a form is created until it is destroyed. Understanding this is key to knowing *where* to put your code.

Key Events in Order:

  1. Load: Fires *before* the form is displayed for the first time. This is the most common place to write initialization code, like populating ListBoxes or setting initial text.
  2. Activated: Fires when the form gains focus (e.g., the user clicks on it).
  3. Shown: Fires *after* the form has been displayed for the first time.
  4. Paint: Fires whenever the form (or part of it) needs to be redrawn.
  5. Deactivate: Fires when the form loses focus (e.g., the user clicks on another application).
  6. Closing: Fires *just before* the form closes (e.g., user clicks the 'X' button). You can *cancel* the close in this event (e.g., to ask "Are you sure you want to exit?").
  7. Closed: Fires *after* the form has been closed and destroyed. Use this for final cleanup.

Unit 4: Web Programming Fundamentals (ASP.NET)

1. What is the role of HTTP Request and HTTP Response in web programming?

They are the two halves of the communication between a client (like your web browser) and a server (the website's host).

  • HTTP Request:
    • A message sent *by the client* (browser) *to the server*.
    • It asks the server to do something, such as:
      • "GET me the contents of /default.aspx" (when you visit a page).
      • "POST this form data to /login.aspx" (when you click a login button).
  • HTTP Response:
    • A message sent *by the server* *back to the client* in reply.
    • It contains the result of the request, such as:
      • The HTML, CSS, and JavaScript for the page ("200 OK").
      • An error message ("404 Not Found").

2. List the main stages of the ASP.NET Page Life Cycle.

The Page Life Cycle is the series of steps an ASP.NET page goes through on the server every time it's requested.

Key Stages (in order):

  1. PreInit: The very first step. Used to create dynamic controls or set the master page.
  2. Init: Controls are initialized and their UniqueIDs are set.
  3. InitComplete: All initialization is finished.
  4. PreLoad: The page's ViewState (old values) and postback data (new values from the form) are loaded into the controls.
  5. Load: The Page_Load event fires. This is the most common place to put code (e.g., set control properties, connect to a database *if not a postback*).
  6. Control PostBack Event(s): The specific event that caused the postback fires (e.g., Button1_Click).
  7. LoadComplete: All loading and events are done.
  8. PreRender: The "last chance" to make any changes to the page or its controls before it gets rendered into HTML.
  9. SaveStateComplete: The final values of the controls are saved to ViewState.
  10. Render: The page and its controls are "drawn" as HTML, which is then sent to the browser.
  11. UnLoad: The very last step. Used for cleanup, like closing database connections or files.

4. Explain... the significance of the runat="server" attribute?

The runat="server" attribute is the magic instruction that connects your ASP.NET page's HTML to the server-side code.

Significance:

  • It tells ASP.NET: "This is not regular HTML. This is a server control."
  • Enables Server-Side Access: It allows you to access and program the control in your C# code-behind (e.g., TextBox1.Text = "Hello";). Without it, TextBox1 wouldn't exist on the server.
  • Enables Events: It allows the control to raise server-side events (like Button1.Click).
  • Maintains State: It allows ASP.NET to automatically save and restore the control's value during a postback (using ViewState).

The <form> tag must have runat="server" for any server controls inside it (like <asp:Button> or <asp:TextBox>) to work.

5. Describe the validation controls in ASP.NET. How do they help in ensuring data integrity?

Validation controls are a set of special controls that automatically check user input. They ensure data integrity by preventing bad or incomplete data from being submitted to the server.

Key Validation Controls:

  • RequiredFieldValidator: Checks that the user did not leave a control (like a TextBox) empty.
  • RangeValidator: Checks that the user's input falls within a specified minimum and maximum (e.g., a number between 1 and 100, or a date in the past).
  • CompareValidator: Compares the input of one control to another (e.g., txtPassword must equal txtConfirmPassword) or to a fixed value.
  • RegularExpressionValidator: Checks that the input matches a specific pattern (a "regular expression"). This is used for emails, phone numbers, postcodes, etc.
  • CustomValidator: Lets you write your own validation logic in code for complex rules (e.g., "check if this username already exists in the database").
  • ValidationSummary: Does not perform validation, but it collects all the error messages from the other validators on the page and displays them in a single, neat list.

Unit 5: Database Connectivity through ADO.NET

1. Differentiate between Connected and Disconnected environments in ADO.NET.

Feature Connected Environment Disconnected Environment
Main Class DataReader (e.g., SqlDataReader) DataSet & DataAdapter (e.g., SqlDataAdapter)
Connection Maintains an open, constant connection to the DB. Connects, gets the data, and immediately disconnects.
Data Access Read-only, forward-only stream of data. Full in-memory copy. Can read, write, update, delete, move forward/backward.

2. What are the key components of the ADO.NET Object Model?

The model is split into two main parts: the .NET Data Provider (the "bridge" to the DB) and the DataSet (the "in-memory" data).

.NET Data Provider Components:

  • Connection (e.g., SqlConnection): Manages the connection to the database (handles the connection string, opening, and closing).
  • Command (e.g., SqlCommand): Represents a SQL statement (SELECT, INSERT, UPDATE, DELETE) or a stored procedure to be executed.
  • DataReader (e.g., SqlDataReader): (Connected model) Reads a high-speed, forward-only, read-only stream of data directly from the database.
  • DataAdapter (e.g., SqlDataAdapter): (Disconnected model) The "bridge" that retrieves data from the DB to Fill a DataSet, and pushes changes from the DataSet back to the DB using Update.

Disconnected Component:

  • DataSet: An in-memory, disconnected copy of the database. It contains DataTables, DataRows, DataColumns, and DataRelations. It has no knowledge of the database it came from.

3. Explain how the DataReader object works in ADO.NET. Why is it used in a connected environment?

The DataReader provides a high-performance, forward-only, read-only "stream" of data from a database.

How it works:

  1. You create a Connection and Command object.
  2. You call connection.Open().
  3. You call command.ExecuteReader(), which returns a SqlDataReader.
  4. You use a while (reader.Read()) loop to advance the reader to the next record. The .Read() method returns true if there is another record.
  5. Inside the loop, you access data for the *current record* (e.g., reader["ColumnName"]).
  6. When finished, you must call reader.Close() and connection.Close().

Why connected? It is the definition of a connected environment because it requires a live, open connection to the database the entire time it is reading. It doesn't copy all the data; it just reads the stream directly from the server, one row at a time.

4. Describe the steps involved in performing CRUD operations using ADO.NET in a connected model.

CRUD = Create (INSERT), Read (SELECT), Update (UPDATE), Delete (DELETE).

For READ (SELECT):

  1. Create SqlConnection and SqlCommand (with SELECT... query).
  2. Open the connection: conn.Open().
  3. Execute the command: SqlDataReader reader = cmd.ExecuteReader().
  4. Loop through results: while (reader.Read()) { ... }.
  5. Close the reader: reader.Close().
  6. Close the connection: conn.Close().

For CREATE, UPDATE, DELETE (INSERT, UPDATE, DELETE):

  1. Create SqlConnection and SqlCommand (with INSERT..., UPDATE..., or DELETE... query).
  2. Open the connection: conn.Open().
  3. Execute the command: int rowsAffected = cmd.ExecuteNonQuery().
  4. (Optional) Check rowsAffected to see if the command was successful.
  5. Close the connection: conn.Close().

5. What is the DataSet class in ADO.NET? Explain how it works in a disconnected environment.

A DataSet is an in-memory database. It's a "disconnected" copy of data retrieved from a database.

It is a collection of DataTables, which in turn are collections of DataRows and DataColumns. It can also hold relationships (DataRelation) between tables.

How it works in a disconnected environment:

  1. A SqlDataAdapter is configured with a SELECT command.
  2. You call adapter.Fill(myDataSet).
  3. The adapter opens a connection, executes the SELECT, fills the DataSet with the data, and immediately closes the connection.
  4. Your application is now "disconnected." You work *only* with the myDataSet object in memory. You can read, change, add, and delete rows. The DataSet tracks all these changes (original vs. current values).
  5. When you are ready to save, you call adapter.Update(myDataSet).
  6. The adapter re-opens the connection, checks all the changes in the DataSet, and automatically generates and executes the required INSERT, UPDATE, and DELETE commands to synchronize the database with your DataSet.
  7. It then closes the connection again.

Unit 6: Building WPF Application

1. Define WPF. What does it stand for and what are its main uses?

  • Stands For: WPF stands for Windows Presentation Foundation.
  • Definition: It is Microsoft's modern UI framework (part of .NET 3.0 and newer) for building rich, visually stunning desktop applications for Windows.
  • Main Uses:
    • Building Windows desktop applications.
    • Creating complex and rich user interfaces that include 2D/3D graphics, animations, documents, and multimedia.
    • It uses XAML (a markup language) to define the "appearance" (the UI) and a code-behind language like C# to define the "behavior" (the logic).

2. What is XAML in WPF? Explain its use in detail.

  • Stands For: XAML stands for Extensible Application Markup Language.
  • Definition: It is an XML-based, declarative language used to create and define the user interface (UI) of a WPF application.
  • Use (Separation of Concerns):
    • XAML allows for a "separation of appearance and behavior."
    • Appearance (The "What"): Designers can define the entire UI—controls, layout, data binding, and styles—in XAML files.
      e.g., <Button Content="Click Me" Width="100" />
    • Behavior (The "How"): Developers can write the application logic (e.g., what happens when the button is clicked) in a separate "code-behind" file (like MainWindow.xaml.cs).
    • This separation allows designers and developers to work on the same application simultaneously.

3. List the five WPF layout panels. Briefly mention the purpose of each.

Layout panels are containers that automatically manage the size and position of the controls inside them.

  1. Grid: The most powerful and common panel. It arranges controls in a flexible grid of rows and columns.
  2. StackPanel: "Stacks" controls one after another in a single line, either vertically (on top of each other) or horizontally (side-by-side).
  3. DockPanel: "Docks" controls to one of the four edges (Top, Bottom, Left, Right). The last control added (by default) fills the remaining center space.
  4. WrapPanel: Arranges controls in a line (horizontally or vertically) and automatically "wraps" to the next line when it runs out of space.
  5. Canvas: The most basic panel. It does *not* automatically arrange controls. You must explicitly position controls using coordinates (e.g., Canvas.Top="10" Canvas.Left="20").

The PasswordBox is a control specifically designed for entering passwords.

5. Describe how animation is implemented in WPF using Storyboards.

WPF has a powerful animation system that works by changing a control's property (like Opacity, Width, or Color) over a set duration.

Storyboard: This is the main "container" or "manager" for animations. A Storyboard holds one or more animations and controls when they start, stop, and how they run.

Implementation Steps:

  1. Create an Animation: Define an animation object for the property you want to change (e.g., a DoubleAnimation to animate a number property like Opacity).
    • Set its From value (e.g., 1.0).
    • Set its To value (e.g., 0.0).
    • Set its Duration (e.g., "0:0:2" for 2 seconds).
    • (Optional) Set AutoReverse="True" or RepeatBehavior="Forever".
  2. Add to Storyboard: Place this animation inside a <Storyboard>.
  3. Set the Target: Tell the animation *what* to animate using attached properties:
    • Storyboard.TargetName="myButton" (the x:Name of the control).
    • Storyboard.TargetProperty="Opacity" (the property to change).
  4. Run the Storyboard: Use an EventTrigger (like Button.Loaded or Button.Click) to Begin the Storyboard.
