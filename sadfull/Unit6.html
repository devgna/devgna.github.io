Unit 6: Designing reliable system, training and conversion

1. Introduction to Software Reliability

  • Definition: A system is reliable if it avoids dangerous or costly failures when used as expected.
  • Levels of Reliability:
    • Meeting Right Requirements: Ensuring the system meets user needs.
    • Reliability of Operations: Ensuring the system delivers expected results without errors.
  • Key Concepts:
    • Error: When the system produces unexpected output.
    • Failure: An error's seriousness and its impact.

2. Approaches to Reliability

  1. Error Avoidance:
    • Prevent errors by identifying user requirements early.
    • Assumes some errors will still occur.
  2. Error Detection and Correction:
    • Detect errors and correct them during use.
    • Example: Fixing a "divide by zero" error.
  3. Error Tolerance:
    • Keep the system functional despite errors.
    • Techniques: Redundant hardware, parallel processing, degraded processing.

3. Training

Importance of Training

  • Facilitates smooth transition from old to new systems.
  • Ensures successful implementation.
  • Addresses changes in user productivity and interactions.
  • Builds confidence and user acceptance.

Components of a Training Program

  • Provide general and specific information about the system.
  • Offer hands-on practice and collect feedback.
  • Allow users to adjust and resolve their queries.
  • Encourage a sense of ownership among users.

User Training

  • Focus on users interacting with the system.
  • Train on:
    • Basic operations (start-up, shutdown, troubleshooting).
    • Data handling (insert, update, delete).
    • Backup and maintenance procedures.

System Operator Training

  • Focus on system upkeep and support.
  • Train on:
    • Installation and operation of the system.
    • Data entry methods.
    • Troubleshooting malfunctions.

Training Methods

  1. Vendor and In-Service Training:
    • Provided by experienced trainers.
    • Covers operation, troubleshooting, and field expert interaction.
  2. In-House Training:
    • Delivered internally with instructional materials.
    • Advantages: Cost-effective and tailored to organizational needs.
    • Disadvantages: Limited hands-on training.

4. Conversion and its Methods

Definition

  • The process of transitioning from the old system to the new system.

Conversion Plans

  • Include activities to implement and operate the new system:
    • List files and data for conversion.
    • Prepare new documents and procedures.
    • Establish controls and cross-checks.
    • Assign responsibilities for activities.
    • Verify schedules.

Conversion Methods

  1. Parallel Conversion: Old and new systems run simultaneously to ensure accuracy.
  2. Direct Conversion: Immediate replacement of the old system with the new one.
  3. Phased Conversion: Gradual implementation in stages, minimizing risks.
  4. Pilot Conversion: Test the new system in a specific area before full implementation.
