Unit 5: Quality assurance Maintenance and Audit

Introduction to Quality Assurance

  • Definition: QA ensures that the system or software meets the required specifications and standards. It reviews and audits software products and related documentation throughout the development life cycle.
  • Purpose: To instill confidence in customers by delivering consistent, reliable, and standard-compliant products.

Objectives of Quality Assurance

  1. Monitor the software development process and final product.
  2. Ensure adherence to standards and procedures.
  3. Notify relevant stakeholders about QA activities and their outcomes.
  4. Address unresolved issues in the software.
  5. Identify and fix deficiencies in products, processes, or standards.

Levels of Quality Assurance

  1. Code Walk-through: Review code for compliance with official coding rules, focusing on documentation and comments.
  2. Compilation and Linking: Ensure the software compiles and links on all platforms and operating systems.
  3. Routine Running: Test software under varying conditions (e.g., event sizes and frequency).
  4. Performance Test: Verify that the software meets predefined performance standards.

Managing Quality Assurance

  • Ensure the system meets specifications for its intended use.
  • Review products and documentation for:
    • Completeness
    • Correctness
    • Reliability
    • Maintainability

Verification & Validation

  • Verification (Alpha Testing): Tests conducted in-house using simulated data to identify errors.
  • Validation (Beta Testing): Real users test the software in a live environment to find issues.

Certification

  • Definition: Certification endorses the correctness of software based on examination and testing.
  • Importance: Certifies that software meets vendor claims, ensuring reliability.

Testing

Definition: Testing evaluates software to detect differences between existing and required conditions, identifying defects or errors.

Key Points:

  • Aims to make the software more robust by uncovering weak areas.
  • Successful tests intentionally expose errors.

Stages of System Testing

  1. Test Strategy: Defines testing levels, methods, tools, and techniques.
  2. Test Plan: Documents testing objectives, tools, responsibilities, and success factors.
  3. Test Case Design: Specifies how individual requirements will be tested and criteria for success.
  4. Test Procedures: Details steps for executing test cases and formats for reporting results.
  5. Test Result Documentation: Summarizes test outcomes, errors found, and overall test success.

Types of Testing

  1. Unit Testing: Tests individual programs or modules independently. Identifies maximum errors compared to other methods.
  2. Integration Testing: Verifies multiple modules working together. Ensures data specifications and file sizes are consistent.
  3. Functional Testing: Tests system functionality against specifications.
    • Positive Testing: Valid inputs produce correct outputs.
    • Negative Testing: Invalid inputs and conditions expose errors.

Rules for System Testing

  • Base testing on user requirements.
  • Understand business logic before writing test scripts.
  • Plan testing early.
  • Use third-party testers.
  • Test for both valid and invalid inputs.
  • Review and document test cases and results.

Control in Information Systems

  • Ensures processes and data are as per design and requirements.
  • Objectives:
    1. Validate input data correctness.
    2. Detect and trace errors.
    3. Ensure legal compliance.
    4. Prevent fraud.

Audit of Information Systems

Definition: A systematic examination of IT infrastructure, practices, and processes to evaluate security, compliance, and efficiency.

Objectives:

  • Identify vulnerabilities.
  • Ensure compliance.
  • Assess control effectiveness.
  • Manage risks.
  • Verify data accuracy.

Maintenance

Definition: Ongoing activities to manage, update, and enhance a system after deployment.

Types:

  • Corrective Maintenance: Fixes errors and bugs.
  • Adaptive Maintenance: Adapts to environmental changes.
  • Perfective Maintenance: Improves performance and functionality.
  • Preventive Maintenance: Identifies and addresses potential future issues.
